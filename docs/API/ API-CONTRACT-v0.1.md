# CTOR API Contract v0.1  
Минимальный API‑контракт для UI, Engine и ИИ‑ботов

---

## 1. Назначение документа

Этот документ определяет минимальный набор форматов и протоколов, необходимых для взаимодействия между всеми модулями CTOR:

- CTOR-UI  
- CTOR-Engine  
- CTOR-AI-AI (JS и Python боты)  
- CTOR-H-AI (человек против бота)  
- CTOR-H-H (человек против человека)  
- CTOR-TEAM-TEAM (командные матчи)

Документ служит **единым стандартом** для всей платформы CTOR и обеспечивает совместимость между модулями.

---

## 2. Основные сущности API

API CTOR оперирует четырьмя ключевыми сущностями:

1. **GameState** — состояние игры  
2. **Move** — ход игрока или бота  
3. **Event** — событие игрового процесса  
4. **Player / Agent** — участник игры  

---

## 3. Формат GameState (обязательный минимум)

GameState — это структура, которую Engine отправляет UI или ботам.

```json
{
  "board": [],
  "current_player": "PLAYER_1",
  "turn": 0,
  "legal_moves": [],
  "score": {},
  "status": "RUNNING"
}
```

### Поля:

| Поле | Тип | Описание |
|------|-----|----------|
| board | массив | Текущее состояние игрового поля |
| current_player | строка | ID игрока, который должен ходить |
| turn | число | Номер хода |
| legal_moves | массив | Список допустимых ходов |
| score | объект | Текущий счёт |
| status | строка | RUNNING / FINISHED / PAUSED |

---

## 4. Формат Move (обязательный минимум)

Move — это структура, которую UI или бот отправляет Engine.

```json
{
  "player_id": "PLAYER_1",
  "move_type": "PLACE",
  "coordinates": [2, 3]
}
```

### Поля:

| Поле | Тип | Описание |
|------|-----|----------|
| player_id | строка | Кто делает ход |
| move_type | строка | Тип хода |
| coordinates | массив | Позиция на поле |

---

## 5. Формат Event

События — это сообщения от Engine к UI или ботам.

Примеры:

```json
{ "event": "MATCH_STARTED" }
```

```json
{
  "event": "MOVE_ACCEPTED",
  "move": { }
}
```

```json
{
  "event": "GAME_OVER",
  "winner": "PLAYER_2"
}
```

---

## 6. Минимальный протокол обмена

### UI → Engine

#### `submitMove(move)`
UI отправляет ход.

---

### Engine → UI

#### `updateGameState(state)`
Engine отправляет новое состояние игры.

#### `sendEvent(event)`
Engine отправляет событие.

---

## 7. Минимальный протокол для ботов (JS и Python)

Боты должны реализовать два метода:

---

### 7.1 Получение состояния игры

```json
{
  "type": "state",
  "payload": { }
}
```

---

### 7.2 Ответ бота — ход

```json
{
  "type": "move",
  "payload": { }
}
```

Это делает JS‑ботов и Python‑ботов полностью совместимыми.

---

## 8. Расширяемость API

API допускает расширение:

- новые типы ходов  
- новые поля в GameState  
- новые события  
- новые режимы игры  

Но **существующие поля изменять нельзя**, чтобы не ломать UI и ботов.

---

## 9. Применение API

| Модуль | Использует API | Роль |
|--------|----------------|------|
| CTOR-UI | Да | UI ↔ Engine |
| CTOR-Engine | Да | Engine ↔ UI / Bots |
| CTOR-AI-AI | Да | Bots ↔ Engine |
| CTOR-H-AI | Да | Human ↔ Bot |
| CTOR-H-H | Частично | UI ↔ Engine |
| CTOR-TEAM-TEAM | Да | Teams ↔ Engine |
