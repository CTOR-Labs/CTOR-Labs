# CTOR API Documentation  
Официальная спецификация API для всех модулей CTOR

Папка **`/docs/API/`** содержит полную, версионированную документацию API, используемую всеми компонентами экосистемы CTOR:

- CTOR‑Engine  
- CTOR‑UI  
- CTOR‑AI‑AI (боты)  
- CTOR‑H‑AI  
- CTOR‑H‑H  
- CTOR‑TEAM‑TEAM  
- CTOR‑Server (сетевой режим)  
- CTOR‑Federation (турниры, рейтинги, международные матчи)

Документы в этой директории являются **единственным источником истины** для всех протоколов CTOR.

---

## 📘 Структура API‑контрактов

Каждая версия API оформлена как отдельный файл:

| Версия | Файл | Описание |
|--------|------|----------|
| **v0.1** | `API-CONTRACT-v0.1.md` | Минимальный API: GameState, Move, Event, базовый протокол UI ↔ Engine |
| **v0.2** | `API-CONTRACT-v0.2.md` | Тайминг, ошибки, расширенные ходы, расширенные события |
| **v0.3** | `API-CONTRACT-v0.3.md` | Сетевой API: HTTP + WebSocket, удалённые клиенты |
| **v0.4** | `API-CONTRACT-v0.4.md` | Авторизация, JWT‑подобные токены, роли, права |
| **v0.5** | `API-CONTRACT-v0.5.md` | Matchmaking, Rooms, Lobbies, очереди |

Каждая новая версия **расширяет** предыдущую, не ломая совместимость.

---

## 🎯 Назначение API

CTOR API определяет:

- формат состояния игры  
- формат ходов  
- формат событий  
- сетевой протокол  
- авторизацию и безопасность  
- взаимодействие UI ↔ Engine  
- взаимодействие Bot ↔ Engine  
- взаимодействие Team ↔ Engine  
- матчмейкинг и лобби  
- управление матчами и комнатами  

API используется всеми модулями CTOR и обеспечивает единый язык взаимодействия.

---

## 🧭 Как использовать API

### 1. Если вы разрабатываете UI  
Используйте:

- v0.1 — базовые структуры  
- v0.2 — ошибки, тайминг  
- v0.3 — WebSocket  
- v0.4 — токены  
- v0.5 — лобби  

### 2. Если вы разрабатываете бота  
Используйте:

- v0.1 — GameState/Move  
- v0.2 — расширенные ходы  
- v0.3 — WebSocket‑клиент  
- v0.4 — токены  

### 3. Если вы работаете над CTOR‑Engine  
Реализуйте **все версии**, начиная с v0.1.

### 4. Если вы создаёте CTOR‑Server  
Основные версии:

- v0.3 — сеть  
- v0.4 — авторизация  
- v0.5 — матчмейкинг  

---

## 🧱 Принципы версионирования

- API развивается **итеративно**  
- каждая версия добавляет новые возможности  
- обратная совместимость сохраняется  
- старые версии не удаляются  
- модули CTOR могут использовать разные версии API
